## 0. TL;DR

The learner model (L3) today is a static ACTFL band with **zero session-state, zero affect**. The research's repeated demand is *"is this student ready to speak right now?"* â€” WTC / anxiety / readiness, not just a CEFR band. S4.1 adds a **heuristic readiness read** derived from the student's recent prior sessions for this assignment (turn-length trend, repair/error density, recent abandonment), snapshots it into `analysis_state['affect_state']`, and uses it to **bias the main tutor's stance gentler when the student is strained** (L5 affect override).

It is the cross-session, **session-start** lever â€” the only one that can change the *main realtime tutor*, since mid-session re-instruction of a voice model is impossible (the same constraint that made S3.3 correction-light a session-start change). It mirrors S2 recycling's proven shape: a fail-open reader over prior sessions â†’ a pure deterministic state â†’ rides `PromptPlan` â†’ modulates the rendered stance. **Byte-identical to today when the flag is off OR the readiness is `neutral`.**

Affect is a **heuristic** read (not a model call â€” "not a smarter model"), exactly like S2 coverage tiers. It **modulates within teacher policy, never overrides it.**

## 1. Scope

### In scope
1. A pure `backend/services/pedagogy/affect.py`: `AffectState` + `compute_affect_state(session_summaries)` (deterministic heuristic) + `affect_stance_lines(affect)` (render directives) + `serialize_affect_state`.
2. An impure fail-open reader `compute_assignment_affect_state(db, bootstrap, uid, assignment_id, *, current_session_id=None)` in `practice_analytics.py`, mirroring `compute_assignment_coverage_state` â€” **gated by `affect_enabled()` (zero reads when off), current-session-excluded, fail-open â†’ None.**
3. `affect_state` carried on `PromptPlan` (parallel to `coverage_state`) and on `analysis_state` (parallel to `coverage`).
4. L5 override: `_build_tutor_stance` consumes `plan.affect` to emit gentler stance lines **only when `readiness == 'strained'`**; byte-identical otherwise.
5. Flag `PEDAGOGY_ENGINE_AFFECT` + `affect_enabled()` gate + cloudbuild wiring (default `'0'`, REPLACE-safe).
6. Tests (pure heuristic table, reader fail-open/gate, render byte-identity, import boundary) + opt-in behavioral eval hook.

### Non-goals (explicitly deferred, documented as follow-ups)
- **Silence-length signal** â€” NOT derivable from existing data (`learning_events.created_at` is server-receipt time, confounded by latency/batching; no client timestamp). Deferred: needs an additive `clientTs`/`silenceMs` capture in the `student.turn` event. Build affect from the three derivable signals first; add silence only if eval shows them insufficient.
- **Within-session live affect â†’ coach-track gating** (soften/suppress S3.2 chips + promote-back when strained). Couples to the not-yet-cut-over S3.2/S3.3 path and cannot touch the main tutor mid-session. Deferred to a follow-up (S4.1b) once chips are cut over.
- **`settled â†’ push harder` lever.** Conservative by design: `settled` means "no softening," not "press the student." Avoids anti-sycophancy / over-correction risk. YAGNI.
- **Free-chat / raw-tutor paths.** Assignment path only (engine scope), like every prior slice.
- **Teacher debrief surfacing of affect** â€” that's S4.2 (L7), which will *read* `analysis_state['affect_state']`.

## 2. Architecture â€” pure / impure split (invariant 7a)

```
PURE   backend/services/pedagogy/affect.py            (stdlib only â€” import-boundary clean)
         â€¢ AffectState{readiness: "settled"|"neutral"|"strained", signals: dict, reason: str}
         â€¢ compute_affect_state(session_signals: list[dict]) -> AffectState
             deterministic heuristic over RECENT prior-session signal dicts (most-recent-first);
             neutral on insufficient/mixed signal (incl. < MIN_SESSIONS_FOR_AFFECT prior sessions).
             Each session_signal = {"avg_words": float, "repair_count": int,
             "turn_count": int, "abandoned": bool} â€” built by the reader from each session
             record (session_summary fields + the record's status). The pure module stays
             ignorant of the session-record shape; it sees only these normalized signals.
         â€¢ affect_stance_lines(affect: AffectState, *, correction_light: bool) -> list[str]
             [] for settled/neutral (byte-identity); gentler stance lines for strained,
             composed so they do not duplicate what correction_light already states
         â€¢ serialize_affect_state(affect: AffectState) -> dict   (for the analysis_state snapshot)

IMPURE backend/services/practice_analytics.py
         â€¢ compute_assignment_affect_state(db, bootstrap, uid, assignment_id, *, current_session_id=None)
             -> AffectState | None
             EXACT mirror of compute_assignment_coverage_state: affect_enabled() gate OUTSIDE
             the try (flag-off â†’ ZERO reads â†’ None); reads prior sessions (current excluded);
             computes via compute_affect_state(summaries); fail-open â†’ None on any error.

GATE   backend/services/pedagogy/integration.py
         â€¢ affect_enabled()  (reads PEDAGOGY_ENGINE_AFFECT; mirrors recycling_enabled/etc.)

PLAN   backend/services/pedagogy/plan.py
         â€¢ compile_prompt_plan(bootstrap, coverage_state=None, affect_state=None) -> PromptPlan
             PromptPlan gains `affect: AffectState | None` (parallel to the coverage field)

RENDER backend/services/pedagogy/render/assignment_prompt.py + backend/services/assignment_resolver.py
         â€¢ render_assignment_prompt(plan, surface, correction_light=False)  (reads plan.affect â€” no new kwarg)
         â€¢ _build_tutor_stance(..., correction_light=False, affect=None)
             appends affect_stance_lines(affect, correction_light=...) when affect.readiness=="strained"
```

**Why affect rides `PromptPlan` (not a render kwarg like `correction_light`):** affect is a *reader-computed state derived from prior sessions* â€” structurally it is `coverage_state`, not a flag boolean. Putting it on the plan keeps the reader â†’ pure-state â†’ render separation identical to S2 and means the pure plan/render layers stay testable without DB.

**Heuristic, not a model call.** Like S2 coverage tiering, affect is a deterministic count/trend read. No second LLM. This is the Â§0 "not a smarter model" discipline and keeps per-session cost at zero new LLM calls.

## 4. The heuristic (`compute_affect_state`)

Reads **recent prior-session signal dicts only** (no `learning_events` needed â€” lighter than coverage; the reader derives each signal dict from a session record's `session_summary` + `status`). "Recent" = the last `AFFECT_WINDOW_SESSIONS` prior sessions, **most-recent-first**.

Three derivable sub-signals, combined into one tier:

| Sub-signal | Source (per session signal) | Strain reading |
|---|---|---|
| **turn_length_trend** | `avg_words` (from `session_summary.average_student_words_per_turn`) | `falling` when the most-recent session's `avg_words` < `TURN_TREND_FALL_RATIO` Ã— (mean `avg_words` of the *preceding* sessions in the window). `unknown` if any input avg is 0/missing or only one session present. |
| **repair_density** | `repair_count / max(turn_count, 1)` per session, where `repair_count = feedback_counts.recast + feedback_counts.elicitation + sum(repeated_error_counts)`, averaged over the window | `high` (window mean `> REPAIR_DENSITY_HIGH`) |
| **abandonment_recent** | session record `status == 'abandoned'` â†’ `abandoned: True` (the reader maps status; `ended_reason` indicating abandonment also counts) | count `>= ABANDONMENT_STRAIN_MIN` within the window |

**Tiering (deterministic, conservative):**
- `neutral` â€” fewer than `MIN_SESSIONS_FOR_AFFECT` (default **2**) prior sessions, OR signals are mixed/insufficient. **This is the byte-identity default** (first/second session = no-op).
- `strained` â€” at least one strong strain signal: `turn_length_trend == "falling"` OR `repair_density == "high"` OR `abandonment_recent >= ABANDONMENT_STRAIN_MIN`. (Disjunctive: any one suffices â€” we err toward gentleness, the low-risk direction.)
- `settled` â€” enough sessions, no strain signals, AND positive engagement (`turn_length_trend in {flat, rising}` and `repair_density == "low"` and zero abandonment). Produces **no** stance change (reserved for S4.2 debrief / future levers).

Constants (tunable, named in the plan, frozen in tests):
`AFFECT_WINDOW_SESSIONS=3`, `MIN_SESSIONS_FOR_AFFECT=2`, `REPAIR_DENSITY_HIGH=0.6`, `ABANDONMENT_STRAIN_MIN=2`, `TURN_TREND_FALL_RATIO=0.7` (latest avg < 0.7 Ã— earlier avg â‡’ falling).

All thresholds are heuristic and **NOT model-verified mastery/affect** â€” documented in LIMITATIONS alongside the S2 coverage caveat.

## 5. L5 override semantics (`affect_stance_lines`)

When `readiness == 'strained'`, append stance lines that make the tutor gentler **within teacher bounds**:

1. **Pacing/patience** â€” allow more productive silence; do not rush; lower follow-up pressure.
2. **Warmth** â€” lead with brief encouragement, acknowledge effort (still no effusive praise â€” anti-sycophancy holds).
3. **Correction softening** â€” *only when NOT correction_light* (when correction_light is on, the coach already owns correction, so this would be redundant): bias to **recast even on rule-based grammar targets**, delay elicitation (the Â§9 matrix row "learner anxiety / low WTC â†’ bias to recast even on grammar," Rassaei). This **respects teacher policy**: an `accuracy_first` teacher's tutor still corrects â€” it just leads with recast and waits longer before escalating.
4. **Turn expectations** â€” accept shorter learner turns; do not demand long production.

`settled` / `neutral` â†’ `[]` (byte-identical).

**Bounded-nudge invariant:** affect never silences correction a teacher asked for, never changes targets, never flips `mode`. It softens *delivery*, not *intent*. (Verified by a test: `accuracy_first` + `strained` still emits a correction directive, just recast-leaning.)

## 6. Integration seams & flow

**Session-start (both surfaces), parallel to S2:**
1. Chat/realtime route resolves the bootstrap, then calls `compute_assignment_affect_state(db, bootstrap, uid, assignment_id, current_session_id=...)`.
2. Result threads into `compile_prompt_plan(bootstrap, coverage_state=..., affect_state=...)` â†’ `PromptPlan.affect`.
3. `render_assignment_prompt(plan, surface, correction_light=...)` â†’ `_build_tutor_stance(..., affect=plan.affect)` appends the strained lines (or nothing).
4. The **session-create** path serializes the same `AffectState` into `analysis_state['affect_state']` via the existing `update_practice_session_analysis_state` writer (NOT `update_practice_session` â€” the S3.1 self-disable lesson).

**Flag-off / neutral â†’ byte-identical** at every seam: reader returns None (zero reads when off), plan.affect is None, render emits no affect lines.

**Independence:** `PEDAGOGY_ENGINE_AFFECT` is standalone â€” no interaction with PROMOTE_BACK/CHIPS/ASK flags. `affect_stance_lines` *reads* `correction_light` only to avoid emitting redundant correction-softening text, not as a gating dependency.

## 7. Error handling

Fail-open everywhere, mirroring S2/S3: any reader/compute/DB error â†’ `affect_state=None` â†’ no stance change, never a 500 on the live chat/realtime path. The flag gate is OUTSIDE the try (flag-off does zero work). Malformed/missing summaries degrade to `neutral`, never raise.

## 10. Invariants (the review lens)

1. **Byte-identical when off or neutral** â€” flag-off (zero reads) and `readiness âˆˆ {neutral, settled}` both render exactly today's prompt.
2. **Bounded nudge, never override** â€” affect softens delivery within teacher policy; never silences requested correction, never changes targets/mode.
3. **Heuristic, no new LLM** â€” deterministic read; zero new per-session model calls.
4. **No new persistence** â€” `affect_state` rides `analysis_state`; one shared fail-open reader (render + snapshot) so safety can't diverge; persisted via `update_practice_session_analysis_state`.
5. **Fail-open** â€” any failure â†’ None â†’ no stance change â†’ never a 500.
6. **Import boundary (7a)** â€” `affect.py` stdlib-only.
7. **Flag gate before reads** â€” `affect_enabled()` outside the try; flag-off does zero work.
8. **Assignment path only** â€” free-chat/raw-tutor untouched.
